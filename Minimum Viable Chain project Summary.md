### 2. **Data Layer (IPFS Integration)**

Decentralized data storage and retrieval through **IPFS**, linking on-chain commitments to off-chain content. This ensures:

* Scalable storage architecture.
* Verifiable, immutable data records.
* Efficient resource usage by keeping large data off-chain.

### 3. **Chain Storage & Communication Logic**

Implements hybrid on-chain/off-chain storage:

* On-chain: metadata, CIDs, state proofs.
* Off-chain: file data, verification hashes, and telemetry streams.
* Uses RPC and REST APIs for seamless developer integration.

### 4. **Interoperability Layer (XCM & Bridge Architecture)**

Provides cross-chain messaging and data collaboration:

* **XCM:** For Substrate-native chains (Polkadot, Kusama, etc.)
* **Bridge Module:** For non-Substrate ecosystems (Ethereum, BSC, Cosmos).
* **Ozed Gateway SDK:** For developers to access and publish cross-chain data.
